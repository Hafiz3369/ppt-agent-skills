# 视觉完成度基准

> 这不是"极限展示"，而是每页的**最低专业标准**。低于此标准的页面等同于草稿。
> 基准的核心理念：**信息丰满 + 视觉层次分明 + 装饰有品位**。

## 5 层视觉架构（每页必须具备）

| 层次 | 要求 | 专业水准 | 不合格表现 |
|------|------|---------|----------|
| 1. 标题区 | 标题(28px+) + 辅助信息(副标题/PART 编号/角标) | 标题有 accent 色关键词高亮，辅助信息提供上下文 | 只有标题，没有任何辅助信息 |
| 2. 内容卡片 | 3-5 张，混合 >= 2 种 card_type | 卡片间有信息层次递进（核心论点 -> 支撑数据 -> 补充说明） | 只有 1-2 张，或全是同类型 |
| 3. 数据可视化 | >= 1 个图表元素 | 图表与数据语义匹配（百分比用环形/进度条，趋势用折线，对比用柱状） | 纯文字叙述数据，没有可视化释放 |
| 4. 装饰层 | >= 2 种技法的组合 | 装饰与内容情绪一致、克制有品位、技法来自工具箱 | 无装饰或装饰堆砌 |
| 5. 页脚 | 品牌/来源/页码 | 与整体风格协调的三段式页脚 | 缺失页脚 |

## 卡片内容密度标准

| 卡片类型 | 最低内容量 | 专业水准的额外加分项 |
|---------|----------|-------------------|
| text | 标题 + >= 2 段正文 + 关键词高亮 | 引用数据佐证论点、有结论性一句话收尾 |
| data | 大数字(36-48px) + 单位 + 趋势 + 解读 + 可视化 | 同比/环比对比、数据来源标注、趋势箭头 |
| list | >= 4 条，每条 15-30 字 | 不同 accent 色圆点、层次递进排列 |
| process | >= 3 步，每步标题 + 描述 + 连线 | 步骤间有视觉节奏变化（大小/颜色渐变） |
| tag_cloud | >= 5 个标签 + >= 2 种 accent | 标签有分组逻辑、主次标签大小有区分 |

## 装饰品位指引

**好的装饰**：
- 服务于信息（如强调线引导视线到核心数据）
- 与风格情绪一致（如学术报告用极简线条而非光晕特效）
- 克制（一页 2-3 种技法足够，多了反而廉价）
- 有层次（背景层 -> 卡片层 -> 内容层各自独立不干扰）

**差的装饰**：
- 与内容无关的纯视觉填充
- 所有页面用完全相同的装饰组合
- 装饰元素过于显眼，抢了内容的视觉权重
- 深色风格的光晕用到浅色风格上

## 页间变化要求

**禁止相邻内容页使用完全相同的视觉结构。** 要求变化的维度：

| 维度 | 变化方式 |
|------|---------|
| 布局 | 相邻页用不同的 layout_hint |
| 装饰重心 | 一页重心在左上，下一页在右下 |
| 卡片构成 | 一页 data 为主，下一页 list+process 为主 |
| 色彩焦点 | 按章节色彩递进规则切换当前章节的 accent 色 |
