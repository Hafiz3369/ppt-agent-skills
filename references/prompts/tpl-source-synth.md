## 格式感知处理指南

不同格式的资料需要不同的处理策略：

| 格式 | 处理方式 | 重点提取 |
|------|---------|---------|
| PDF | 用 pdf-reader 或 markitdown 解析 | 图表数据、结论段落、脚注引用 |
| Word (.docx) | 用 markitdown 转换后读取 | 章节结构、核心论点、嵌入表格 |
| Excel (.xlsx) | 用 markitdown 转换后读取 | 数据表格→直接转为结构化数据包格式 |
| PPT (.pptx) | 用 markitdown 提取文本和结构 | 每页标题+要点、speaker notes、数据图表 |
| 纯文本/Markdown | 直接读取 | 按语义分块提取 |
| 代码文件 | 直接读取 | 架构说明、注释、README 中的描述 |
| 图片 | 用图像识别描述内容 | 图表数据、截图内容、品牌元素 |

## 执行流程

1. 读取 `{{REQUIREMENTS_PATH}}` 了解资料使用策略（全量吸收 / 择优引用 / 仅背景参考 / 严格基于原文）
2. 读取或遍历 `{{SOURCE_INPUT}}` 路径中的所有资料文件，**按上方格式感知指南选择处理方式**
3. 按 Playbook 的结构化提炼方法处理每份资料
4. 综合所有资料，输出 `source-brief.txt` 到 `{{BRIEF_OUTPUT}}`
5. **主动提取 PPTX 友好型结构化数据**（与 research 分支的 search-brief.txt 保持同构格式，见下方产物格式要求）
6. 自审：确认覆盖所有必含内容、标注资料边界、不夸大
7. 发送 FINALIZE

## source-brief.txt 产物格式（强制，与 search-brief.txt 同构）

source-brief.txt 必须包含以下区块，使下游大纲和策划阶段能以统一方式消费：

```text
# 资料摘要
## 基本信息
- 来源类型：用户现有资料（非联网搜索）
- 资料文件数：N
- 覆盖主题：...

## 核心论点与数据
（按主题分组，每条数据标注来源文件名）
### [主题1]
- [论点/数据] -- 来源：[文件名]

## 关键数据清单
- [数据] -- 来源：[文件名] -- 置信度：高/中/低

## PPTX 结构化数据包
（必须包含，至少覆盖 3 种数据类型；从用户资料中主动提取，格式同 search-brief.txt）

### metrics
- {value} {unit} ({trend}) [来源: {文件名}]

### data_tables
| 维度 | ... | ... |
[来源: {文件名}]

### timelines
{time}: {milestone}
[来源: {文件名}]

### expert_quotes
"{quote}" -- {person}, {org} [来源: {文件名}]

### process_flows
1. {step_title}: {desc}
[来源: {文件名}]

### 数据覆盖评估
- metrics: N 个可用 / 缺口: ...
- comparisons: N 个可用 / 缺口: ...
- timelines: N 个可用 / 缺口: ...

## 素材边界
- 覆盖完整的内容：...
- 资料中缺失的内容（PPT 可能需要但资料未覆盖）：...
- 数据矛盾项（如有）：...

## 资料使用建议
- 强论据（直接引用）：...
- 辅助背景（间接参考）：...
- 不建议引用（置信度低或有争议）：...
```

## 硬规则

- 不做联网搜索，只处理用户已提供的资料
- 对引用的具体数据必须标注来源文件名
- 不得补充用户资料中没有的信息（严格模式下）
- 若资料中有矛盾数据，如实标注而非取一
- **数字精确记录**，不概括为"增长了很多"（必须是"同比增长 47.3%"）
- **source-brief.txt 必须包含独立的 `## PPTX 结构化数据包` 区块**，至少覆盖 3 种数据类型
- 完成后发送 FINALIZE，由主 agent 回收并关闭你
