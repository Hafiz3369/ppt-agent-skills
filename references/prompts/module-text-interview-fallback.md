# Text Fallback Mode -- 结构化文本采访单

## 核心指令

当前 CLI 不支持原生结构化采访 UI，必须回退为**结构化文本采访单**。

禁止退化为：

- 一整行 `场景=；受众=；目标动作=...`
- 没有分组的散乱追问
- 只问 3-4 个字段的缩水版本

## 固定输出格式

```markdown
请按下面结构回复，可以直接填写选项序号（如 A、B）；不确定可以选“默认”；也可自行补充说明。

**A. 场景与目标**
- 场景：【A. 内部汇报】 【B. 产品发布/宣讲】 【C. 招商合作/融资路演】 【D. 培训课件】 【E. 个人介绍】 【F. 其他：___】
- 受众：【A. 团队内部/高层】 【B. 客户/合作方】 【C. 技术同行】 【D. 大众/外部用户】 【E. 其他：___】
- 目标动作：【A. 建立认知】 【B. 促成决策/投资】 【C. 愿意加入/传播】 【D. 仅完成汇报】 【E. 其他：___】

**B. 内容与边界**
- 页面密度：【A. 少而精(字少图大)】 【B. 适中(图文并茂)】 【C. 信息量大(干货密集)】
- 必含内容：【自由补充】
- 必避内容：【自由补充】
- 资料策略与分支：【A. research (联网扩写)】 【B. 非 research (仅依体现有资料)】

**C. 风格与素材**
- 风格：【A. 极简商务】 【B. 科技极客】 【C. 数据驱动/严谨】 【D. 自动匹配主题】 【E. 其他：___】
- 品牌限制：【自由补充，如: 主色调蓝】
- 语言：【A. 中文】 【B. 英文】 【C. 中英混排】
- 配图：【A. decorate (仅装饰配图)】 【B. generate (AI绘图)】 【C. provided (只用给定图)】 【D. manual_slot (留空槽位)】
- 封面署名信息：【A. 不需要】 【B. 需要（请补充：姓名/职位/日期等）】

**D. 成功标准与执行**
- 成功标准：【自由补充】
- 子代理模型：【A. 跟主代理一致】 【B. 指定模型：___】
```

若需要极简回法，可在结尾补一句：

```text
如果你想省事，也可以直接回复：全部按默认，用 research。
```

写盘时仍需在 `interview-qa.txt` 里补一段 canonical 锚点，例如：

```text
## 归纳后的问答落点
- target_action: ...
- must_avoid: ...
- material_strategy: research
```
