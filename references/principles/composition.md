# 构图与留白

> 留白不是"浪费空间"，而是信息的呼吸。好的构图引导视线，差的构图让视线迷路。

## 格式塔原则

人脑会自动把视觉元素组织为有意义的整体：

| 原则 | 含义 | PPT 应用 |
|------|------|---------|
| **接近性** | 靠近的元素被视为一组 | 卡片内部 gap 小（8-12px），卡片之间 gap 大（20px） |
| **相似性** | 外观相似的元素被视为同类 | 同级卡片用相同圆角/阴影/内边距 |
| **连续性** | 视线沿线条/方向流动 | 时间线/流程图的节点用连线引导 |
| **闭合性** | 人脑自动补全不完整的形状 | 卡片边框可以用半透明描边（暗示边界但不硬切） |
| **图底关系** | 前景(图)和背景(底)要清晰分离 | 卡片背景 vs 页面背景至少有明暗差 |

## 三分法

将画布横纵各三等分，重要元素放在交叉点或线上：

```
┌───┬───┬───┐
│   │   │   │  交叉点是视觉强点
├───┼───┼───┤  标题放上 1/3 线
│   │   │   │  核心数据放中心或右上交叉点
├───┼───┼───┤  页脚放下 1/3 线
│   │   │   │
└───┴───┴───┘
```

- 不要所有内容都居中（除非是章节封面/金句页）
- 左上角是自然阅读起点，重要信息从这里开始

## 留白的三个层级

| 层级 | 位置 | 作用 |
|------|------|------|
| **宏观留白** | 页面边距（40px） | 把内容框在安全区内，防止贴边 |
| **中观留白** | 卡片间距（20px gap） | 区分不同的信息组 |
| **微观留白** | 卡片内部间距（8-16px） | 区分标题/正文/数据/图表 |

**留白递增法则**：层级越高的间距越大。卡片内部间距 < 卡片之间间距 < 页面边距。如果三者大小接近，层级感就消失了。

## 自检

- 页面是否有明确的"呼吸空间"（不是每寸都塞满）？
- 去掉所有文字后，布局骨架本身是否清晰有序？
- 三个层级的留白是否递增（微观 < 中观 < 宏观）？
