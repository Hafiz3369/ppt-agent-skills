# 视觉层级与 CRAP 原则

> 人眼不是逐字阅读，而是按视觉权重跳跃扫描。设计的核心任务是用视觉手段控制阅读顺序。

## CRAP 四原则（Robin Williams）

### Contrast（对比）
- **原理**：差异越大，层级越清晰。相似的元素会被视为同级
- **PPT 应用**：标题 vs 正文至少差 2 级字号（如 28px vs 14px）；强调数据用 accent 色而非加粗；不同层级的卡片用不同视觉重量
- **常见错误**：标题和正文字号只差 2px（看不出层级）；所有文字都加粗（等于没加粗）

### Repetition（重复）
- **原理**：重复使用相同视觉元素创造统一感和可预测性
- **PPT 应用**：所有页面使用同一套 CSS 变量；卡片标题前统一用 3px accent 竖线；页脚格式跨页一致
- **常见错误**：每页用不同的装饰风格（看起来像拼凑）；某些页有页脚某些没有

### Alignment（对齐）
- **原理**：页面上没有任何元素是随意放置的，每个元素都应与其他元素有视觉连接
- **PPT 应用**：使用 CSS Grid 严格对齐（而非绝对定位猜坐标）；卡片间距统一（20px gap）；左对齐是最安全的选择
- **常见错误**：居中对齐混用左对齐（形成锯齿边缘）；元素之间没有网格约束

### Proximity（亲密性）
- **原理**：相关的信息靠近，不相关的信息远离。间距传达关系
- **PPT 应用**：标题-正文间距 16px（紧密），卡片间距 20px（区分）；数据数字和标签间距 8px（数字和标签是一对）
- **常见错误**：所有元素间距一样（分不清谁属于谁）；太多信息挤在一起没有分组

## 视觉重量阶梯

视觉权重从高到低：

```
超大数字(64-80px) > 页面标题(28px) > 配图 > 卡片标题(18px) > 数据数字(36px) > 正文(14px) > 标注(12px)
```

**一页只有一个视觉锚点**：最先吸引视线的元素。如果两个元素争夺锚点地位（如两个 48px 数字并排），观众不知道先看哪个。

## 自检

- 闭上眼再睁开，第一个看到的元素是否是这页最重要的信息？
- 能否在 3 秒内识别出页面的层级结构（标题 -> 主内容 -> 辅助）？
- 跨页翻阅时，视觉系统是否一致（不是每页一种风格）？
