# 风格系统

## 核心理念

> **8 个预置风格是参考调色板，不是固定制服。** 它们提供经过验证的色彩搭配和装饰思路，但你完全可以：
> - 基于预置风格微调色值（换一个更深的蓝、更暖的灰）
> - 混搭两种风格的特质（blue_white 的结构感 + luxury_purple 的质感）
> - 根据主题完全自创配色方案（只需遵守下方的色彩原则）
>
> **唯一不可突破的底线**：所有颜色必须通过 CSS 变量引用，确保全局一致性。

## 风格数据模型

每套风格的 CSS 变量结构（自创风格时也必须提供完整的变量集）：

```css
:root {
  /* 背景层 */
  --bg-primary: #主背景;
  --bg-secondary: #渐变过渡色;
  /* 卡片层 */
  --card-bg-from: #卡片渐变起;
  --card-bg-to: #卡片渐变止;
  --card-border: rgba(边框色);
  --card-radius: 12px;
  /* 文字层 */
  --text-primary: #标题/正文;
  --text-secondary: rgba(辅助文字);
  /* 强调色（4 个，用于章节色彩递进） */
  --accent-1: #主强调;
  --accent-2: #次强调;
  --accent-3: #第三强调;
  --accent-4: #第四强调;
}
```

## 色彩原则（自创配色时的硬底线）

| 原则 | 要求 | 目的 |
|------|------|------|
| 对比度安全 | 文字与背景的对比度 >= 4.5:1 | 确保可读性 |
| 60-30-10 法则 | 背景色 60%、卡片/辅助色 30%、accent 色 10% | 避免"花哨" |
| accent 克制 | 同一页 accent 色不超过 2 种（不同页可以不同） | 视觉焦点清晰 |
| 深浅一致 | 深色背景配浅色文字，浅色背景配深色文字 | 基本可读性 |
| 卡片与背景区分 | 卡片背景与页面背景有可感知的色差（至少 5% 明度差） | 卡片边界清晰 |

## 风格决策流程

### 决策优先级

| 优先级 | 信号 | 行动 |
|--------|------|------|
| 1 | 用户**明确指定**了风格或品牌色 | 直接用 / 基于品牌色自创 |
| 2 | 主题有强烈的情绪倾向 | 选择匹配情绪的调色板，可微调 |
| 3 | 受众有明确偏好 | 参考下方受众映射 |
| 4 | 均无明显倾向 | blue_white 作为安全默认 |

### 受众 -> 情绪 -> 调色板

| 受众 | 期望的情绪 | 参考调色板 |
|------|----------|----------|
| 企业管理层/投资人 | 专业可信 | blue_white / minimal_gray / 自创冷色系 |
| 政府/事业单位 | 稳重正式 | blue_white / royal_red / 自创庄重色系 |
| 学术/研究人员 | 冷静克制 | minimal_gray / 自创低饱和度色系 |
| 开发者/技术团队 | 极客未来感 | dark_tech / 自创深色科技系 |
| 年轻消费者 | 活泼创意 | vibrant_rainbow / 自创亮色系 |
| 高端客户 | 奢华质感 | luxury_purple / 自创深色高级系 |

### 自创配色指引

当预置调色板都不完美匹配时，可以自创：

1. 确定**情绪关键词**（如"温暖专业"、"冷酷科技"、"自然清新"）
2. 选择一个最接近的预置调色板作为**起点**
3. 按以下策略微调：
   - 换 accent 色 -> 保持背景和文字不变，只调强调色
   - 换色温 -> 冷色系换暖色系（蓝->橙）或反之
   - 换明度 -> 浅底换深底（同时翻转文字色和卡片色）
4. 输出完整的 CSS 变量集（必须覆盖上述所有变量）

## 装饰技法工具箱

> **装饰不是固定清单，是可自由组合的技法。** 每种技法有管线安全等级和适用场景。每页至少使用 2-3 种技法的组合。

### 背景层技法

| 技法 | 实现方式 | 适合的情绪 | 管线安全 |
|------|---------|----------|---------|
| 网格点阵 | `radial-gradient(circle, color 1px, transparent 1px)` + `background-size` | 科技、精密 | 安全 |
| 渐变色块 | 大面积 `div` + `radial-gradient` + 低 opacity | 柔和、自然 | 安全 |
| 光晕效果 | `div` + `radial-gradient` + accent 色 5-8% opacity | 未来感、发布会 | 安全 |
| 几何线条 | `div` 细线条 + 角度旋转 | 建筑、工程 | 安全 |
| 纹理底图 | `<img>` + opacity 0.03-0.08 | 文化、质感 | 安全 |

### 卡片层技法

| 技法 | 实现方式 | 效果 |
|------|---------|------|
| 左侧强调线 | `div` 3-4px 宽, accent 色 | 视觉锚点 |
| 编号气泡 | `div` 32-40px 圆形, accent 背景 | 序号感 |
| 顶部色带 | `div` 4px 高, 渐变色, 卡片顶部 | 分类标记 |
| 图标装饰 | 内联 SVG 图标 + 圆形底色 | 语义增强 |
| 卡片悬浮感 | `box-shadow` 多层阴影 | 层次感（仅浅色风格） |

### 页面层技法

| 技法 | 实现方式 | 效果 |
|------|---------|------|
| 角标装饰 | SVG `path` L形线 / 品牌标记 | 精致感 |
| 分隔渐隐线 | `div` 1px, `linear-gradient` 渐隐 | 区域划分 |
| 大号水印数字 | `div` 120-160px 数字, opacity 0.03-0.05 | 页面辨识度 |
| 标题下划线 | `div` 4px 高, 40-80px 宽, accent 渐变 | 标题强调 |
| 品牌色块 | `div` 大面积圆角矩形, accent 低 opacity | 风格标记 |

### 管线安全底线（所有技法必须遵守）

- 所有装饰必须是**真实 DOM 节点**（`div` / `span` / 内联 `svg`）
- 禁止 `::before` / `::after` 伪元素
- 禁止 `mask-image`、`conic-gradient`、CSS border 三角形
- 图形优先用内联 SVG
- 详细禁止清单见 `pipeline-compat.md`

## 多语言排版优化

### 英文场景

| 层级 | 中文默认 | 英文调整 | 原因 |
|------|---------|---------|------|
| H0 封面标题 | 48-56px | 44-52px | 英文字母视觉比中文汉字小 |
| H1 页面标题 | 28px | 26px | 同上 |
| Body 正文 | 13-14px | 13px | 英文 13px 已够可读 |
| 行高 | 1.8 | 1.6 | 英文不需那么大的行高 |
| 字间距 | 默认 | `letter-spacing: 0.01em` | 微调提升可读性 |

英文字体推荐栈：
```css
font-family: 'Inter', 'SF Pro Display', 'Segoe UI', system-ui, -apple-system, sans-serif;
```

### 日韩语言

| 语言 | 字体推荐 |
|------|---------|
| 日文 | `'Hiragino Kaku Gothic ProN', 'Yu Gothic', 'Meiryo', system-ui, sans-serif` |
| 韩文 | `'Apple SD Gothic Neo', 'Malgun Gothic', 'Nanum Gothic', system-ui, sans-serif` |

排版规则与中文相同（CJK 字符共享类似的排版特征）。
