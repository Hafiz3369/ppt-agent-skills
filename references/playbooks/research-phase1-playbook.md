# ResearchSynth Phase 1 Playbook -- 搜索规划与执行

## 目标

多维度检索、主动狩猎 PPTX 友好型的结构化数据素材，输出最丰富的原始网页片段与事实，为下一阶段整理做准备。你不负责整理格式或生成摘要，只负责**深挖线索、提取精确数字与事实**。

---

## 搜索维度规划（至少覆盖以下全部 6 大维度）

| 维度 | 示例查询 | 目的 |
|------|---------|------|
| **核心定义** | "{主题} 是什么 / 定义 / 核心概念" | 基础事实 |
| **市场数据** | "{主题} 市场规模 / 增长率 / 行业报告 2024-2026" | 数据填充 |
| **竞品/对比** | "{主题} vs {竞品} / 对比分析" | 对比论证 |
| **案例/应用** | "{主题} 客户案例 / 应用场景" | 故事化说服 |
| **趋势/展望** | "{主题} 发展趋势 / 未来展望" | 结尾素材 |
| **权威观点** | "{主题} 专家评价 / 行业报告 / 白皮书" | 权威背书 |

### 动态查询规划基准（按目标页数）

| 页数规模 | 推荐查询总数 |
|---------|------------|
| <= 8 页 | 5-8 |
| 9-18 页 | 10-15 |
| > 18 页 | 15+ |

## PPTX 结构化数据狩猎目标（核心）

搜索不是泛泛浏览网页。你在提炼段落时，必须刻意寻找以下 11 种可被下游排版为图表/卡片的强结构化数据。一旦发现，精确记录数值、单位、时间和来源，不得用“增长显著”概括跨度：

| 数据类型 | 搜索猎物长什么样 |
|---------|-----------------|
| `metrics` | 具体的 KPI、年营收、月活、转化率、市场占有率 |
| `data_tables` | 包含多项参数的复杂对比表格、规格表 |
| `trend_series`| 历年增长柱状数据、过去五年的演变节点 |
| `ranked_list` | 行业 Top 5、最受欢迎的 N 大工具榜单 |
| `before_after`| 客户采用前后的对比数值、成本下降百分比 |
| `funnel_data` | 整个周期的留存率、各环节流失率 |
| `pie_data`    | 营收构成占比、用户群画像比例（必须加总接近 100%）|
| `timelines`   | 完整的发展里程碑、公司发展编年史 |
| `expert_quotes`| 真实行业专家的原话、研究员/CEO引言（必须带人名）|
| `team_profiles`| 核心创始人背景、关键专页人物资历 |
| `process_flows`| 典型的 1-2-3 操作步骤、架构闭环说明 |

## 执行边界（Phase 1 专有）

1. **先列计划**：把你想出的多维度查询词打印到控制台，再发起工具请求。
2. **服从轮次上限**：总搜索轮次不得超过任务包中给定的 `MAX_SEARCH_ROUNDS`；在上限内优先覆盖缺口最大的维度。
3. **直检 + 扩检**：首次并行搜索后，如发现某些指标（如营收）缺失，立刻补充 1-2 次长尾深挖检索。
4. **只原样摘取**：将有价值的原始文本、数字和出处（带 URL 溯源）原样拿回。
5. **禁止前置整理**：不要在本阶段尝试去重、不要写 brief 摘要（那是下一阶段的工作）。
