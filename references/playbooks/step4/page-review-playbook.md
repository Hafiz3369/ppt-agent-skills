# Page Visual Review & Fix Playbook -- 单页图审与 HTML 修复

## 目标

截图后充当**极度严苛、像素敏感的资深前端架构与 UI 设计总监**，对刚生成的 HTML 进行多维度扫雷，发现问题立刻亲手修改源码，直到页面真正达到可交付标准。

---

## 审查维度（10 维，每轮必须全部过检）

截图后，针对以下每个维度输出：`[通过]` 或 `[发现: 具体描述]`，不得跳过任何一条。

---

### 维度 1 — 画布边界合规（硬红线，不通过则整页不通过）

扫描：
- `body` 是否声明 `width: 1280px; height: 720px; overflow: hidden`？
- 是否有任何内容（文字、色块、卡片）超出截图可见边界，被硬生生截断？
- 是否有 `transform: scale()` 缩放 hack？

修复：
- 超出边界的元素：用 `max-height` + `overflow: hidden` 拉回，或缩小字号
- 有 scale hack：删除 hack，重新按 1280x720 计算布局

---

### 维度 2 — 对比度可读性

扫描：
- 白色/浅色文字放在了浅色背景上（无法看清）？
- 深色文字放在了深色背景上（糊在一起）？
- 渐变背景的过渡区域出现了文字与背景颜色接近的区段？
- 数据数字字号是否比普通正文大至少 8px（突出数据）？

修复：
- 加 `text-shadow: 0 1px 3px rgba(0,0,0,0.6)` 或改写背景色
- 加半透明遮罩：`background: rgba(0,0,0,0.45)` 或 `rgba(255,255,255,0.35)`
- 改写 CSS 变量使文字与背景达到至少 4.5:1 对比度

---

### 维度 3 — 排版密度与留白

扫描：
- 元素是否贴着 1280x720 外边框（外层 padding < 40px）？
- 卡片之间间距是否 < 12px（太拥挤）？
- 行高是否 < 1.4（文字挤在一起）？
- 内容区是否占满全屏没有任何留白（视觉疲劳感强）？

修复：
- 增大外层 `padding` 至 40px
- 增大卡片 `gap` 至 16-24px
- 设置 `line-height: 1.5` ~ `1.7`
- 非必要内容考虑删除，保留核心信息，增加呼吸感

---

### 维度 4 — Z 轴遮挡与层叠事故

扫描：
- 装饰性几何形状/SVG 是否遮住了核心文字或数字？
- 两个并列卡片是否发生了重叠或阴影叠加？
- 绝对定位的元素是否意外覆盖了其他内容？

修复：
- 内容层：`z-index: 10`+
- 装饰层：`z-index: 1`，`opacity: 0.08~0.15`
- 检查所有 `position: absolute` 元素，确认不遮挡内容

---

### 维度 5 — 字体渲染质量

扫描：
- 字体是否降级为 Times New Roman 或默认黑体（Google Fonts 加载失败）？
- 各级标题字号是否有清晰梯度（至少 4px 差距）？
- 正文字号是否 < 12px（在截图中接近不可读）？
- 中英文混排时是否出现字体不一致的割裂感？

修复：
- 检查 `<style>` 中的 `font-family` 是否正确继承 `var(--font-primary)`
- 确认 Google Fonts 的 `@import` 地址无误
- 正文最小字号 14px，标题与正文差距不低于 6px

---

### 维度 6 — 内容完整性（对照 planning）

扫描：
- 读取 `planning{n}.json` 的 `cards`，逐一确认每张卡片的 `title` 和 `body` 都在 HTML 中出现？
- 大纲中标注的关键数据是否都已展示？
- 是否有卡片的 `body` 内容被 `overflow: hidden` 截断成看不到的片段？

修复：
- 缺失的内容补回，或缩小字号使其完整可见
- 过长的文字截断：改用 `overflow: hidden; text-overflow: ellipsis; display: -webkit-box` 方案，或做字号自适应

---

### 维度 7 — 图片/装饰合理性

扫描：
- 若 `image.mode=decorate`：装饰元素（SVG/色块/渐变）是否与页面主题调性一致，还是风马牛不相及？
- 装饰物是否抢过了核心内容的视觉焦点（太大太亮）？
- 若 `image.mode=generate/provided`：图片是否正常加载显示（无破图图标）？

修复：
- 过大的装饰元素缩小或降低透明度
- 调性不符的装饰替换为更恰当的几何图形或渐变色块
- 破图：修正 `src` 路径

---

### 维度 8 — 视觉重心与焦点感

扫描：
- 页面是否有一个明确的视觉焦点（`focus_zone` 指定的区域）？
- 是否所有元素大小相近、没有主次之分（"大锅饭"布局）？
- 页面的视觉动线是否从左上 → 主焦点 → 辅助信息流畅延伸？

修复：
- primary 卡片：字号增大 20%，加 `border-left: 4px solid var(--accent-1)` 或 `box-shadow`
- secondary 卡片：字号保持，降低背景色饱和度
- 添加几何装饰引导视觉进入焦点区

---

### 维度 9 — 色彩统一性

扫描：
- 页面用色是否超过 4 种（不含灰阶）且感觉杂乱？
- 所有颜色是否来自 `style.json` 的 `css_variables`，没有硬编码颜色值（如 `#ff0000`）？
- 卡片背景、文字、强调色之间是否有协调感，还是互相冲突？

修复：
- 替换所有硬编码颜色为 CSS 变量
- 过多颜色：统一到 `--accent-1`/`--accent-2` 两种强调色
- 增加颜色之间的过渡（渐变或饱和度梯度）

---

### 维度 10 — 设计感与品质感

扫描（主观但有标准的扫描）：
- 看起来像"普通前端页面"还是"高端演示文稿"？
- 是否有设计细节传递质感（微妙阴影、圆角一致性、字重对比、精准对齐）？
- 与 `style.json.design_soul` 描述的情绪是否吻合？
- `signature_move` 是否出现在页面中作为跨页识别锚点？

修复（若不达标）：
- 添加 `box-shadow: 0 4px 20px rgba(0,0,0,0.12)` 提升卡片质感
- 调整所有 `border-radius` 至统一值（如 12px 或 16px）
- 大标题加 `letter-spacing: -0.02em` 提升紧致感
- 添加 `signature_move` 对应的视觉元素

---

## 闭环承诺

- **看图 → 逐维度输出结构化报告 → 亲手改代码 → 截新图 → 确认 → 继续**
- 任何"我认为应该没问题"都不算，必须截图验证
- 最多 3 轮，但坚决不交带溢出问题的稿件
- 3 轮后仍有小瑕疵：尽力修复并在 FINALIZE 中如实报告剩余问题
