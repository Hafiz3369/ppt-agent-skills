# Image Review & Code Fix Playbook

## 目标
执行截图脚本，充当**极度严苛、像素敏感的资深前端架构与UI设计总监**。如果存在违和、溢出、低级错误，立刻手动修改 HTML 文件进行抢救，并重新截图直至达标为止。

## 五大严苛图片自审维度（像素级扫雷）

每次得到 PNG 后，请张大眼睛对照以下 5 条“死线”进行审查，发现任何一条不合规，立刻修改 HTML 源码（不是口头指出，是**亲自修改**）并再截一张看效果：

### 1. 物理红线溢出与裁切 (Overflow & Crop)
- **绝对要求**：整个 PPT 基于 1280x720 盒子。
- **排雷**：任何文字、卡片、背景色块是否超出了截图的绝对边缘被硬生生砍掉一半？文字段落是否冲出了容器？如果有，立刻去 HTML 里用 `max-height` / `overflow: hidden` 或者缩小字体将其拉回安全区。

### 2. 对比度死穴 (Contrast Clash)
- **排雷**：白色文字是不是放在了同样高亮的背景图或色块上导致根本看不清？深色文字有没有放在了深色暗角里糊成一团？
- **修复**：如果是，去 HTML 里给背后的元素加渐变遮罩 `background: rgba(0,0,0,0.5)`，或者加上 `text-shadow`、改写色值变量。必须达到投屏级的高清晰度。

### 3. 排版拥挤与留白缺失 (Margin & Padding Panic)
- **排雷**：文字行距是否太小挤在一起（低于 1.4）？元素是不是直接贴着 1280x720 的外边框（没有 40px 的外层 padding）？卡片与卡片之间是不是叠在一起？
- **修复**：扩大 HTML 里面的 margin / padding / line-height。增加大量留白，呼吸感是高级设计的唯一捷径。

### 4. 层叠遮挡事故 (Z-Index Overlap)
- **排雷**：绝对定位的装饰形状、图标是不是把核心文字遮住了？或者并列的两张卡片发生重影？
- **修复**：排查 HTML 的 `position: absolute`，调整 `z-index` 把文字挪到最上层，把无关紧要的装饰物推向底层或降低不透明度（opacity: 0.1）。

### 5. 样式坍塌 / 降级兜底 (Style Fallback Check)
- **排雷**：看起来像没写 CSS 的毛坯房？字体变成了丑陋的 Times New Roman 或默认黑体？
- **修复**：检查并修复有没有把 `<style>` 写破，是否遗漏了 `var(--font-primary)` 从 Style 合同中对齐的定义。

## 闭环承诺
- 看图 -> 找刺 -> 改代码 -> 截新图 -> 确认 -> 结束。
- 单页最多自我博弈打磨 3 轮。如果依然有小瑕疵，尽力抢救后可 FINALIZE 放行，但坚决不交带溢出的事故稿。
